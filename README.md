# EDTANet
Evidential Disentanglement and Semantic-Visual Topological Alignment for Cross-domain Plant Disease Classification
This repository implements the method proposed in the research on cross-domain plant disease classification, which leverages evidential disentanglement and semantic-visual topological alignment to address domain shift issues in plant disease recognition tasks.
The directory structure and core file/folder functions are detailed below:
├── dataset/                # Dataset-related modules
│   ├── __init__.py
│   ├── Adataset.py      # Data loading pipeline for cross-domain plant disease datasets
│   └── config.py    # Domain configuration files (define source/target domains, data paths, etc.)
│   └── description.json    # Description of diseases generated by gpt-4 
├── utils/                  # Utility functions
│   ├── __init__.py
│   ├── loss.py   # Definition of custom loss functions (e.g., evidential loss, topological alignment loss)
│   └── test_visual.py    # Visualization tools for test results
│   └── utils.py    # Definition of logger 
├── load_text.py            # Generate text-based .pth files (semantic features/text embeddings for plant diseases)
├── network.py              # Definition of the core network architecture (evidential disentangler, topological alignment module)
├── train.py                # Main training script (training loop, parameter optimization, validation)
└── README.md               # Project documentation
**Dataset Preparation**
Organize your cross-domain plant disease dataset following the structure specified in dataset/config.py:
datasets/
├── source_domain/        # e.g., PlantVillage (source domain)
│   ├── Apple_scab/
│   ├── Tomato_Bacterial_spot/
│   └── ...
└── target_domain/        # e.g., PlantDoc (target domain)
    ├── Apple_scab/
    ├── Tomato_Bacterial_spot/
    └── ...
**Running the Code**
Step 1: Generate Text Embedding File
First, generate the text-based .pth file (semantic features for plant disease classes):python load_text.py
Step 2: Train the Model
Run the training script with domain configuration:python train.py --source_domain source_domain --target_domain target_domain
Step 3: Visualize Test Results
After training, use the visualization tools in utils/ to analyze results.
